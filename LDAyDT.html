4. Se importa la librera numpy y funcin DecisionTreeClassifier.

Se define una lista de valores de alpha, que es el parmetro de poda. Se prueban los valores de alpha en validacin cruzada siendo, una metodologa de LOOCV, para encontrar cal produce el mejor desempeo. Esto significa que se entrena y evala el modelo con validacin cruzada y se obtiene el desempeo promedio en todas las particiones, guardando el resultado en scores.

Utlizando np.argmax(scores) se selecciona el primer valor de alpha que se encuentra con el score mximo. Este alpha ser de utilidad para la poda del rbol. Se prueban 3 valores distintos de alpha con 10 folds y evalua cual de estos es el ms adecuado de todos.

Teniendo el valor de alpha ptimo se genera y entrena el rbol podado para posteriormente visualizar el resultado final. Es relevante mencionar que para controlar la poda del rbol se utiliza ccp_alpha lo cual conlleva el mtodo de Cost Complexity Pruning, donde mayor sea ccp_alpha, ms nodos se eliminarn, generando un rbol ms simple, por lo que si llega a valer cero no se hara poda y el rbol se expandera por completo.

Respecto al resultado del rbol de decisin, en la parte del nodo raz se tiene como condicin Final grade <= 9.5 con un ndice gini indicando cierto grado de impureza, 385 estudianres que no abandonaron y 69 que s abandonaron. Para la divisin de los dos nodos, del del lado izquierdo tiene un ndice gini muy bajo, indicando que casi todos los ejemplos pertenecen a la clase de los que s abandonaron (69 estudiantes), con esto es posible decir que si un estudiante tiene un final grade <= 9.5 es casi seguro que abandonar (69 de 70 lo hicieron). Por el del lado derecho, hay un ndice gini de cero, indicando que todos pertenecen a los que no abandonaron, por lo que si un estudiante tiene final grade mayor a 9.5 es seguro que no abandonar.

Solamente se usa la variable Final grade puesto que el rbol indica que est es ms fuerte para predecir la desercin, si una variable ya es suficiente para dividir correctamente las clases, el rbol no requiere ms divisiones.

In[13]:
import numpy as np
from sklearn.tree import DecisionTreeClassifier as DTC, plot_tree

alphas=[0.1, 0.15, 0.2]  
kf=StratifiedKFold(n_splits=10)

scores=[cross_val_score(DTC(ccp_alpha=alpha), X_train_selected, y_train_selected, cv=kf).mean() for alpha in alphas]

best_alpha=alphas[np.argmax(scores)]
print(f"Mejor valor de alpha seleccionado: {best_alpha}")

pruned_tree=DTC(ccp_alpha=best_alpha).fit(X_train_selected, y_train_selected)

plot_tree(pruned_tree, feature_names=X_train_selected.columns)
plt.title("rbol de Decisin Podado")
plt.show()

Conclusin

Con base a los modelos obtenidos especficamente para este trabajo, se logra identificar diferencias en las mtricas calculadas para cada uno de estos.

  • Exactitud (Accuracy): el rbol de decisin tiene mejor exactitud (99.48%) que el LDA (90.26%), clasificando correctamente ms observaciones en total.
  • Precisin: el rbol de decisin tiene mejor precisin (96.67%) que el LDA (91.67%), indicando que hace menos falsas alarmas.
  • Sensibilidad: el rbol de decisin tiene una sensibilidad ms alta (100%) que el LDA (37.93%). En este mtrica es la que ms diferencia hay entre un modelo y otro, y esto se debe a que se mide cuntos de los casos positivos reales fueron correctamente identificados por el modelo. En el caso del LDA estudiantes que realmente abandonaron son clasificados errneamente como no abandono, aumentando los falsos negativos.
  • F1-score: el rbol de decisin es mejor (98.3%) que el LDA (53.65%). Al estar relacionado con la sensibilidad, (esta mtrica es el equilibrio entre precisin y sensibilidad) el f1 score del LDA se ve directamente afectado.
  • Especificidad: ambos tienen la misma especificidad con 99.39%.

En cuanto a la matriz de confusin, el rbol de decisin detecta correctamente a los 29 estudiantes que abandonaron (TP) con 0 FN, mientras que LDA solo detecta 11 (TP) con 18 FN. En este caso se ve que LDA pierde 18 estudiantes que s abandonaron.

Debido a que en todas las mtricas evaluadas el rbol de decisin tiene resultados ms altos, para este caso en especfico, el mejor modelo es el rbol de decisin. Al ser todas las mtricas altas, surge una gran sospecha de sobre ajuste, sin embargo es posible observar que la profundidad del rbol ni el nmero de hojas es grande, por lo que debido a que el rbol es simple parece ser muy efectivo. No obstante es relevante buscar considerar otros factores para tener una prediccin ms completa.
